# Security Policy

## Supported versions

This project is small and shell-first. Security fixes are expected to land on
the current default branch and be consumed from the latest published revision.

## Reporting a vulnerability

Do not report undisclosed vulnerabilities in public issues or pull requests.

Use GitHub Security Advisories or GitHub Private Vulnerability Reporting for
this repository when available. If private reporting is not enabled yet, delay
public disclosure until a private reporting path has been established with the
maintainer.

When reporting a vulnerability, include:

- affected version or commit
- operating system and OpenSSH version
- exact command line or config that triggers the issue
- expected behavior and observed behavior
- minimal reproduction steps

Do not include private keys, passphrases, PINs, or other live credentials in
reports.
